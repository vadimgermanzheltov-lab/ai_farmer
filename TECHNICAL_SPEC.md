# ТЗ для Codex: демо-версия AI-консультанта «Умный птицевод»

## 1. Цель проекта
Создать демо-версию B2C AI-консультанта для начинающих владельцев домашней птицы. Консультант помогает выбрать вид птицы, рассчитать базовые параметры содержания, подобрать кормление и выдать итоговые рекомендации.

Проект учебный. История диалогов, личный кабинет, база знаний на бэкенде и сохранение прогресса пользователя не требуются.

## 2. Название
Рабочее название: **Умный птицевод**.

## 3. Формат продукта
Одностраничное веб-приложение с диалоговым окном и быстрыми плашками-ответами.

Основной принцип общения:
- 70% взаимодействия через кнопки/плашки;
- 30% — свободный текстовый вопрос;
- пользователь проходит короткие сценарии;
- в конце получает структурированный блок рекомендаций.

## 4. Технологическая схема
Рекомендуемый стек для Codex:
- React + Vite;
- локальные JSON-файлы в папке `/src/data` или `/public/data`;
- API-запрос к OpenRouter из отдельного файла-сервиса;
- ключ OpenRouter хранить в `.env`.

Пример переменной окружения:
```env
VITE_OPENROUTER_API_KEY=your_key_here
VITE_OPENROUTER_MODEL=openai/gpt-4o-mini
VITE_SITE_URL=http://localhost:5173
VITE_SITE_NAME=AI Farmer Demo
```

OpenRouter использует OpenAI-compatible endpoint:
`https://openrouter.ai/api/v1/chat/completions`

Обязательный заголовок:
`Authorization: Bearer <OPENROUTER_API_KEY>`

Опциональные заголовки:
- `HTTP-Referer`
- `X-OpenRouter-Title`

## 5. Главное ограничение безопасности
AI-консультант не должен ставить диагнозы и назначать лечение птице. При симптомах болезни он может дать чек-лист первичной проверки и рекомендовать обратиться к ветеринарному специалисту.

Запрещено:
- назначать лекарства;
- указывать дозировки препаратов;
- ставить диагноз;
- обещать гарантированный результат.

## 6. Основные сценарии

### 6.1. Стартовый экран
Показать приветствие:
«Здравствуйте! Я AI-Фермер. Помогу выбрать птицу, рассчитать условия содержания, подобрать кормление и разобраться с частыми проблемами.»

Кнопки:
- Выбрать птицу
- Рассчитать хозяйство
- Подобрать кормление
- Проблемы и болезни
- Я новичок
- Задать свой вопрос

### 6.2. Сценарий «Выбрать птицу»
Цель: подобрать вид птицы и примерные породы/кроссы.

Вопросы:
1. Для какой цели?
   - яйца
   - мясо
   - яйца и мясо
   - продажа
2. Какой опыт?
   - новичок
   - есть опыт
   - фермер
3. Сколько голов?
   - до 10
   - 10–30
   - более 30
4. Есть ли теплое помещение?
   - да
   - нет
   - планирую построить

Итог:
- рекомендуемый вид птицы;
- подходящие породы/кроссы;
- почему подходят;
- что подготовить до покупки.

### 6.3. Сценарий «Рассчитать хозяйство»
Цель: рассчитать базовые параметры содержания.

Вопросы:
1. Вид птицы:
   - куры
   - утки
   - гуси
   - индюки
   - перепела
2. Количество голов:
   - 10
   - 20
   - 50
   - 100
   - другое
3. Цель:
   - для семьи
   - для продажи
   - тестовый запуск

Итог:
- площадь помещения;
- площадь выгула;
- ориентировочный корм в день и месяц;
- количество гнезд/насестов, если применимо;
- стартовый чек-лист оборудования.

### 6.4. Сценарий «Подобрать кормление»
Цель: дать базовую схему кормления без ветеринарных назначений.

Вопросы:
1. Вид птицы.
2. Возраст:
   - птенцы
   - молодняк
   - взрослая птица
3. Цель:
   - яйцо
   - мясо
   - поддержание здоровья

Итог:
- основные компоненты рациона;
- частота кормления;
- что обязательно должно быть всегда;
- частые ошибки.

### 6.5. Сценарий «Проблемы и болезни»
Цель: помочь пользователю разобраться с распространенной проблемой без диагноза.

Вопросы:
1. Что происходит?
   - перестали нестись
   - тонкая скорлупа
   - плохо едят
   - вялые
   - медленно растут
   - другое
2. Возраст птицы.
3. Изменялись ли корм, свет, температура или условия содержания?

Итог:
- возможные бытовые причины;
- что проверить в первую очередь;
- когда срочно обращаться к ветеринару;
- предупреждение, что это не диагноз.

### 6.6. Сценарий «Я новичок»
Цель: выдать простой план запуска.

Вопросы:
1. Что хотите получить?
   - домашние яйца
   - мясо для семьи
   - небольшой заработок
   - попробовать мини-ферму
2. Где будете содержать?
   - дача
   - частный дом
   - деревня/ферма
3. Сколько готовы начать?
   - 5–10 голов
   - 10–30 голов
   - больше 30

Итог:
- рекомендуемый старт;
- план подготовки на 30 дней;
- минимальный список покупок;
- ошибки новичков.

## 7. Структура итоговой рекомендации
Финальный ответ должен быть структурирован:

1. Краткий вывод.
2. Рекомендуемый вариант.
3. Расчет/параметры.
4. Что подготовить.
5. Частые ошибки.
6. Следующий шаг.
7. Предупреждение по ветеринарным вопросам, если тема связана со здоровьем птицы.

## 8. UI-требования
Интерфейс:
- область сообщений;
- плашки быстрых ответов;
- поле для свободного ввода;
- кнопка «Начать заново»;
- финальный блок «Ваш план».

Стиль:
- спокойный фермерский визуал;
- простые карточки;
- зеленые/земляные оттенки;
- крупные кнопки;
- без перегрузки текстом.


## 8.1. Визуальное оформление и изображения

В приложении должны быть предусмотрены изображения, но без перегрузки интерфейса. Для учебной демо-версии использовать легкие SVG-иллюстрации из папки `/public/images`. Позже их можно заменить на реальные фотографии пород, курятников и оборудования.

Где использовать картинки:
- стартовый экран: одна hero-картинка с фермерской тематикой;
- сценарий «Выбрать птицу»: карточки видов птицы с изображением, названием и кратким описанием;
- итоговый блок «Ваш план»: 1–2 изображения — подходящий тип птицы и вариант курятника/выгула;
- сценарий «Рассчитать хозяйство»: карточка курятника, выгула, кормушек/поилок.

Ограничения:
- не больше 3 изображений на одном экране;
- изображения должны быть декоративными и не мешать чтению рекомендаций;
- у каждого изображения должен быть `alt`;
- путь к изображениям брать из `data/visual_assets.json`;
- если изображение не загрузилось, показывать текстовую карточку без ошибки интерфейса.

Рекомендуемый вид карточки:
```text
[изображение]
Название: Куры
Кратко: простой старт для яиц и домашнего хозяйства
Кнопка: Выбрать
```

## 9. Файлы данных
Использовать JSON-файлы:
- `app_config.json` — название, стартовые кнопки, дисклеймеры;
- `poultry_catalog.json` — виды птицы, назначение, плюсы/минусы;
- `housing_norms.json` — базовые нормы площади, корма и оборудования;
- `scenarios.json` — сценарии, вопросы, варианты ответов;
- `recommendation_templates.json` — шаблоны итоговых рекомендаций;
- `safety_rules.json` — правила безопасности ответов.

## 10. Логика работы
1. Пользователь выбирает сценарий.
2. Приложение показывает вопросы из `scenarios.json`.
3. Ответы сохраняются только в текущем состоянии страницы.
4. После последнего вопроса приложение берет нормы из `housing_norms.json` и данные из `poultry_catalog.json`.
5. Формирует промпт для OpenRouter.
6. AI возвращает итоговую рекомендацию.
7. Пользователь может начать новый расчет.

## 11. Требования к промпту AI
AI должен:
- отвечать простым языком;
- не перегружать пользователя;
- использовать данные из локальных JSON как приоритет;
- не выдумывать точные цифры, если их нет в базе;
- формулировать ориентировочные значения как «примерно», «ориентировочно»;
- не давать ветеринарных назначений.

## 12. Критерии готовности
Демо считается готовым, если:
- работает стартовый экран;
- есть минимум 5 сценариев;
- пользователь может пройти сценарий через кнопки;
- формируется финальная рекомендация;
- API-ключ OpenRouter подключается через `.env`;
- при отсутствии ключа показывается демо-ответ по шаблону;
- есть базовые JSON-файлы.
